## 1. 프로젝트 개요

### 1.1 프로젝트 정보

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 스마트 물류관리 시스템 구축 |

| **계약금액** | 15억원 (VAT 별도) |
| **계약기간** | 2025-01-06 ~ 2025-12-31 (총 12개월) |

### 1.2 프로젝트 배경 및 목적

글로벌물류는 연간 500만건 이상의 물류를 처리하는 국내 Top 5 물류기업으로, 현재 레거시 시스템의 노후화와 급증하는 물동량에 대응하기 위해 차세대 스마트 물류관리 시스템 구축이 필요한 상황입니다.

- **현황 및 문제점**: 10년 이상 운영된 레거시 시스템으로 인한 처리 속도 저하, 실시간 재고 파악 불가, 타 시스템 연동 한계
- **추진 목표**: AI 기반 물류 최적화, 실시간 재고/배송 추적, 클라우드 기반 확장성 확보
- **기대 효과**: 물류 처리 효율 30% 향상, 재고 정확도 99.5% 달성, 운영 비용 20% 절감

### 1.3 프로젝트 범위

#### 1.3.1 업무 범위

| 구분 | 범위 | 비고 |
|------|------|------|
| **포함 범위** | 입고/출고 관리, 재고 관리, 배송 추적, 정산 시스템 | 4개 핵심 모듈 |
| **제외 범위** | 회계 시스템, HR 시스템, 기존 ERP 시스템 | 인터페이스만 개발 |

#### 1.3.2 시스템 범위

| 시스템 구분 | 대상 시스템 | 주요 기능 |
|------------|------------|-----------|
| WMS | 창고관리시스템 | 입출고, 재고, 피킹/패킹 |
| TMS | 배송관리시스템 | 배차, 경로최적화, 추적 |
| OMS | 주문관리시스템 | 주문접수, 할당, 정산 |
| Dashboard | 통합모니터링 | 실시간 현황, KPI, 리포트 |

## 2. 추진 전략 및 방법론

### 2.1 추진 전략

본 프로젝트는 "안정적 전환"과 "혁신적 기능 구현"의 균형을 핵심 가치로 추진합니다.

**핵심 전략:**
1. **단계적 전환 전략**: Big Bang 방식이 아닌 모듈별 순차 전환으로 리스크 최소화
2. **애자일 기반 개발**: 2주 단위 스프린트로 빠른 피드백과 유연한 대응
3. **현업 밀착 협업**: 주 2회 현업 리뷰를 통한 요구사항 정합성 확보

### 2.2 수행 방법론

| 항목 | 내용 |
|------|------|
| **방법론명** | Agile-Waterfall 하이브리드 |
| **선정 사유** | 대규모 SI 프로젝트 안정성 + 애자일 유연성 결합 |

**단계별 주요 활동:**
- **분석**: 요구사항 정의, 현행 분석 (산출물: 요구사항정의서, AS-IS 분석서)
- **설계**: 아키텍처/UI/DB 설계 (산출물: 설계서 일체)
- **개발**: 기능 구현, 단위 테스트 (산출물: 소스코드, 단위테스트결과서)
- **테스트/이행**: 통합/인수 테스트, 시스템 전환 (산출물: 테스트결과서, 이행계획서)
